# 컴퓨터 이해 & 프로그래밍 기초 — 시험 중요 포인트 정리

## 1. 데이터(Data)와 정보(Information)의 차이 ⭐ 중요

### 왜 중요한가?

* 강의 초반 핵심 개념으로 반복 설명
* 컴퓨터의 역할과 연결되는 기본 개념
* 이후 프로그래밍/컴퓨터 구조 이해의 기반

### 데이터(Data)

* 현상에 대한 관찰·측정 결과
* 질적 데이터: 붉다, 밝다, 뜨겁다
* 양적 데이터: 3500도 등 수치 표현

### 핵심 정리

* 데이터 = 관찰/측정 결과
* 질적 데이터 = 정성적 표현
* 양적 데이터 = 수치 표현

---

## 2. 정보(Information) 정의 ⭐ 중요

### 왜 중요한가?

* 데이터와 반드시 구분해서 이해해야 한다고 설명
* 시험에서 정의형 문제 가능성 높음

### 정보란?

* 문제 해결에 사용할 수 있도록 가공된 데이터 집합
* 기존 데이터를 처리해서 만들어진 결과

### 정보 생성 방법

* 정렬
* 계산
* 군집화
* 표현 방식 변경

### 핵심 정리

* 데이터 → 처리/가공 → 정보
* 정보는 “가공된 데이터”

---

## 3. 컴퓨터의 정의 ⭐ 중요

### 왜 중요한가?

* 컴퓨터 개론의 가장 기본 개념
* 정의형 문제 가능성 높음

### 컴퓨터 정의

> 정의된 방법에 따라 입력된 데이터를 자동 처리하여 정보를 생산하는 기계

### 핵심 정리

* 입력: 데이터
* 처리: 프로그램/연산
* 출력: 정보

---

## 4. 비트(Bit)와 바이트(Byte) ⭐ 시험 핵심

### 왜 중요한가?

* 매우 중요하게 설명
* 컴퓨터가 데이터를 표현하는 기본 단위
* 시험 단골 개념

---

### 비트(Bit)

* 데이터를 표현하는 최소 단위
* 0과 1 두 상태만 표현 가능

### 특징

* 1비트 → 2가지 상태
* 2비트 → 4가지 상태
* 3비트 → 8가지 상태

### 핵심 공식

* 표현 가능한 경우의 수 = 2ⁿ

---

### 바이트(Byte)

* 8개의 비트 묶음
* 1Byte = 8bit

### 특징

* 2⁸ = 256가지 표현 가능

### 핵심 정리

* 비트 = 최소 단위
* 바이트 = 실제 데이터 처리 기본 단위

---

## 5. 트랜지스터와 이진법 ⭐ 매우 중요

### 왜 중요한가?

* 컴퓨터가 왜 0과 1만 사용하는지 설명
* “컴퓨터와 이진법은 떨어질 수 없다” 강조

### 핵심 내용

* 트랜지스터는 ON/OFF 두 상태만 표현 가능
* 이를 0과 1로 사용

### 핵심 정리

* 트랜지스터 → 이진법 사용 원인
* 컴퓨터는 이진수 기반으로 동작

---

## 6. 숫자의 표현 방식 ⭐ 중요

### 왜 중요한가?

* 진법 변환 직접 설명
* 계산 문제 가능성 있음

### 핵심 내용

* 컴퓨터는 이진수만 처리 가능
* 십진수 ↔ 이진수 변환 필요

### 시험 포인트

* 이진수 각 자리:

  * 2⁰
  * 2¹
  * 2²
  * 2³ ...

### 핵심 정리

* 십진수 → 2로 계속 나눔
* 이진수 → 각 자리값 계산

---

## 7. 문자 표현과 인코딩 ⭐ 중요

### 왜 중요한가?

* 숫자와 문자 처리 차이 강조
* ASCII 시험 자주 나옴

### 인코딩이란?

* 문자와 비트 패턴을 서로 대응시키는 체계

### 대표 인코딩

* ASCII(아스키 코드)

### 핵심 정리

* 문자는 진법 변환 불가능
* 문자 ↔ 이진수 매핑 필요
* 이를 인코딩 체계라고 함

---

## 8. 데이터 타입이 필요한 이유 ⭐ 중요

### 왜 중요한가?

* 강사가 “잘 기억해 두라”고 언급

### 핵심 내용

* 숫자와 문자 모두 이진수로 저장됨
* 따라서 이 값이 숫자인지 문자인지 구분 필요

### 핵심 정리

* 데이터 타입 = 데이터 종류 구분 목적

---

## 9. 하드웨어와 소프트웨어 차이 ⭐ 시험 가능성 높음

### 하드웨어

* 물리적인 컴퓨터 부품

### 소프트웨어

* 하드웨어를 동작시키는 프로그램

### 핵심 정리

* Hardware = 만질 수 있음
* Software = 프로그램

---

# 10. 폰 노이만 구조 ⭐⭐⭐ 시험 최중요

### 왜 중요한가?

* 매우 길고 자세히 설명
* “거의 모든 컴퓨팅 장치가 사용” 강조

### 폰 노이만 구조 5대 장치

1. 입력 장치
2. 기억(저장) 장치
3. 제어 장치
4. 연산 장치
5. 출력 장치

### 핵심 정리

* 현재 대부분의 컴퓨터 구조 기반

---

## 11. 입력 장치 / 출력 장치 ⭐ 중요

### 입력 장치

* 명령과 데이터를 컴퓨터에 전달

### 예시

* 키보드
* 마우스
* 터치스크린
* 스캐너

---

### 출력 장치

* 처리 결과를 사람이 이해 가능한 형태로 출력

### 예시

* 모니터
* 스피커
* 프린터
* HUD
* HMD

---

## 12. 기억 장치와 저장 장치 차이 ⭐ 시험 자주 나옴

### 기억 장치(RAM)

#### 특징

* 빠름
* 비쌈
* 용량 작음
* 전원 차단 시 데이터 소실

---

### 저장 장치(HDD, SSD)

#### 특징

* 느림
* 용량 큼
* 전원 꺼져도 데이터 유지

### 핵심 정리

| 구분    | 기억장치   | 저장장치 |
| ----- | ------ | ---- |
| 속도    | 빠름     | 느림   |
| 가격    | 비쌈     | 저렴   |
| 전원 차단 | 데이터 소실 | 유지   |

---

## 13. CPU 구조 ⭐ 중요

### CPU 역할

* 제어 + 연산 수행

### CPU 구성 요소

* 제어 유닛(CU)
* 연산 논리 유닛(ALU)
* 레지스터

### 핵심 정리

* CPU = 제어장치 + 연산장치 패키징

---

## 14. 메모리(RAM) 경유 과정 ⭐ 매우 중요

### 왜 중요한가?

강사가:

> “상당히 중요하니까 꼭 기억해 두시길 바랍니다.”

라고 직접 강조함.

### 핵심 내용

* 보조기억장치(HDD/SSD) → RAM → CPU 순서로 이동
* HDD에서 CPU로 직접 가지 않음

### 핵심 정리

반드시:

> 저장장치 → RAM → CPU

순서로 처리

---

## 15. 알고리즘 정의 ⭐ 시험 핵심

### 왜 중요한가?

* 매우 길게 설명
* 정렬 예제로 반복 설명

### 알고리즘이란?

> 문제 해결을 위한 단계별 절차

### 핵심 특징

* 추상화된 문제 해결 방법
* 여러 상황에 공통 적용 가능

---

## 16. 선택 정렬(Selection Sort) ⭐ 중요

### 왜 중요한가?

* 정렬 알고리즘 예제로 직접 설명

### 핵심 과정

1. 가장 작은 값 선택
2. 맨 앞 값과 교환
3. 정렬 완료 여부 확인
4. 반복

### 핵심 정리

* 선택 정렬 = 최소값 선택 후 앞으로 이동

---

## 17. 프로그래밍 언어 세대 ⭐ 시험 가능성 높음

### 1세대 언어

* 기계어
* 0과 1 직접 사용

### 2세대 언어

* 어셈블리어

### 3세대 언어

* Python
* C
* Java

### 4세대 언어

* 자동화 중심 언어

### 핵심 정리

* 1·2세대 = 저급 언어
* 3·4세대 = 고급 언어

---

## 18. 컴파일러 vs 인터프리터 ⭐ 중요

### 컴파일러

* 전체 번역 후 실행
* 책 번역 비유 사용

### 인터프리터

* 실시간 한 줄씩 번역
* 통역 비유 사용

### 핵심 정리

| 구분 | 컴파일러  | 인터프리터  |
| -- | ----- | ------ |
| 방식 | 전체 번역 | 실시간 번역 |
| 비유 | 번역가   | 통역가    |

---

## 19. 시스템 소프트웨어 종류 ⭐ 중요

### 종류

1. 컴파일러
2. 운영체제(OS)
3. 유틸리티

### 운영체제 역할

* 하드웨어 관리
* 응용 프로그램 실행 환경 제공

### 핵심 정리

* OS는 하드웨어를 효율적으로 관리
