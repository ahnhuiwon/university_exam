## 1. 실전 프로젝트 2 개요 ⭐⭐⭐ 중요

### 프로젝트명
> 틱택토(Tic-Tac-Toe) 게임

### 핵심 목표
- 이벤트 기반 프로그램의 개념 이해
- 게임 프로그램 구조 이해
- 클래스를 이용한 프로그램 설계
- 다음 강의에서 실제 구현할 기능 설계

---

# 2. 헐리우드 법칙 ⭐⭐⭐ 중요

### 핵심 문장
```text
Don't call us, we'll call you.
```

### 의미
- 사용자가 계속 프로그램을 직접 호출하는 방식이 아님
- 프로그램이 이벤트가 발생했을 때 반응하는 방식

---

# 3. 제어의 역전 ⭐⭐⭐ 시험 핵심

### 정의
> 프로그램 흐름의 주도권이 사용자 코드가 아니라 시스템/프레임워크로 넘어가는 것

### 핵심 정리
- 기존 방식: 사용자가 명령을 직접 실행
- 이벤트 기반 방식: 사용자가 이벤트를 발생시키면 프로그램이 반응

---

# 4. 이벤트 기반 프로그래밍(EDP) ⭐⭐⭐ 시험 핵심

### 정의
> 이벤트를 중심으로 프로그램이 동작하는 프로그래밍 방식

### EDP
```text
Event Driven Programming
```

### 핵심 정리
- 사용자의 입력이나 사건에 따라 프로그램이 반응
- 게임, GUI 프로그램에서 중요

---

# 5. 이벤트(Event) ⭐⭐⭐ 중요

### 정의
> 프로그램이 감지하고 처리할 수 있는 사용자의 동작이나 사건

### 예시
- 키보드 입력
- 마우스 클릭
- 조이스틱 이동
- 타이머 신호
- 다른 프로그램의 신호

---

# 6. 배치 프로그램(Batch Program) ⭐⭐⭐ 중요

### 정의
> 정해진 데이터를 정해진 순서대로 일괄 처리하는 프로그램

### 핵심 정리
- 입력 데이터 → 프로그램 처리 → 정보 파일 생성
- 실행 흐름이 미리 정해져 있음

---

# 7. 배치 프로그램 vs 이벤트 기반 프로그램 ⭐⭐⭐ 시험 핵심

| 구분 | 배치 프로그램 | 이벤트 기반 프로그램 |
|---|---|---|
| 실행 흐름 | 미리 정해짐 | 이벤트에 따라 달라짐 |
| 입력 시점 | 실행 전/처리 중 고정 | 사용자가 발생시킴 |
| 예시 | 데이터 분석, 파일 변환 | 게임, GUI 앱 |

---

# 8. 이벤트 기반 프로그램 구조 ⭐⭐⭐ 시험 핵심

## 주요 구성

| 구성 요소 | 의미 |
|---|---|
| 이벤트 발생 장치 | 이벤트를 만드는 장치 |
| 이벤트 큐 | 이벤트가 쌓이는 공간 |
| 이벤트 핸들러 | 이벤트를 처리하는 코드 |
| 이벤트 루프 | 이벤트를 계속 확인하는 반복 구조 |

---

# 9. 이벤트 큐(Event Queue) ⭐⭐⭐ 중요

### 정의
> 발생한 이벤트가 순서대로 쌓이는 공간

### 핵심 정리
- 이벤트는 즉시 처리되지 않을 수 있음
- 큐에 쌓인 뒤 순서대로 처리됨

---

# 10. 이벤트 핸들러(Event Handler) ⭐⭐⭐ 시험 핵심

### 정의
> 특정 이벤트가 발생했을 때 실행되는 코드

### 예시
- 스페이스바 입력 → 점프
- 마우스 클릭 → 선택
- 버튼 클릭 → 화면 전환

---

# 11. 이벤트 루프(Event Loop) ⭐⭐⭐ 시험 핵심

### 정의
> 이벤트가 발생했는지 계속 확인하고 처리하는 반복 구조

### 핵심 정리
```text
이벤트 확인
↓
이벤트 가져오기
↓
이벤트 처리
↓
다시 이벤트 확인
```

---

# 12. 게임 루프(Game Loop) ⭐⭐⭐ 시험 핵심

### 정의
> 게임이 실행되는 동안 계속 반복되는 핵심 흐름

## 기본 흐름

```text
게임 초기화
↓
이벤트 검사
↓
이벤트 처리
↓
게임 상태 업데이트
↓
화면 업데이트
↓
반복
```

---

# 17. 틱택토 게임 ⭐⭐⭐ 시험 핵심

### 정의
> 3x3 게임판에 두 플레이어가 번갈아 기호를 놓고, 가로/세로/대각선을 먼저 완성하면 승리하는 게임

---

# 18. 틱택토 기본 규칙 ⭐⭐⭐ 중요

1. 두 명의 플레이어가 번갈아 둔다.
2. 한 명은 O, 다른 한 명은 X를 사용한다.
3. 빈칸에만 둘 수 있다.
4. 가로, 세로, 대각선 중 하나를 같은 기호로 채우면 승리한다.
5. 빈칸이 없고 승자가 없으면 무승부다.

---
