# 4강 - 구조적 프로그래밍

## 1. 고우트문(GOTO)과 데이크스트라 ⭐ 매우 중요

### 중요도
⭐⭐⭐⭐⭐

### 왜 중요한가?
- 구조적 프로그래밍의 시작점
- 시험/면접 단골 개념

### 핵심 정리
1968년 에츠허르 데이크스트라(Edsger Dijkstra)가 발표한 논문:

```text
"GOTO Statement Considered Harmful"
(고우트문은 해롭다)
````

### 핵심 주장

* GOTO문은 프로그램 흐름을 복잡하게 만든다.
* 유지보수를 어렵게 만든다.
* 예측 불가능한 코드가 된다.

## 4. 구조적 프로그래밍 패러다임 ⭐ 핵심

### 중요도

⭐⭐⭐⭐⭐

### 왜 중요한가?

* 이번 강의 핵심
* 시험 가능성 매우 높음

### 핵심 정리

* GOTO 없이 프로그램 작성
* 세 가지 구조만 사용

### 세 가지 구조

| 구조    | 의미        |
| ----- | --------- |
| 순차 구조 | 순서대로 실행   |
| 선택 구조 | 조건에 따라 실행 |
| 반복 구조 | 반복 실행     |

### 시험 포인트

* 구조적 프로그래밍 = 순차 + 선택 + 반복
